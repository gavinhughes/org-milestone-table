#+title: Changelog

* <2026-04-16 Thu>
- Feature: ~org-milestone-table-highlight-critical-path~ custom variable (default ~nil~) controls whether ~C-c C-c~ automatically highlights the critical path. Use ~org-milestone-table-toggle-critical-path~ to toggle independently at any time.
- Feature: ~org-milestone-table-critical-path~ face now has explicit light/dark theme color specs (amber palette) instead of inheriting ~highlight~, making it straightforward to customize via ~M-x customize-face~.

* <2026-04-15 Tue>
- Feature: Critical-path highlighting — after ~org-milestone-table-update-timeline~, rows on the critical path are highlighted with ~org-milestone-table-critical-path~ face.
- Feature: ~org-milestone-table-toggle-critical-path~ command — toggle critical-path overlay highlighting on/off without recomputing dates.
- Feature: Undated rows are topologically sorted by predecessor relationships and spliced immediately before their earliest dated dependent, rather than being placed at the end.
- Bugfix: Silently skip undated predecessors during date resolution instead of raising an error.
- Bugfix: ~C-c C-c~ now preserves point position after updating and sorting the table.

* <2026-04-01 Wed>
- Feature: Validation errors are displayed in a dedicated ~*org-milestone-table errors*~ buffer rather than the minibuffer.
- Feature: Duplicate milestone IDs are now detected and reported as errors.
- Feature: Fuzzy milestone IDs (~NNN?~) — a trailing ~?~ marks a milestone as a placeholder; fuzzy IDs are excluded from circular-reference detection and sorted after their base ID.

* <2026-03-29 Sun>
- Feature: ~org-milestone-table-dwim~ bound to ~C-c C-c~ via ~org-ctrl-c-ctrl-c-hook~; runs add-missing-IDs, update-dates, and sort-by-date in sequence.
- Feature: ~org-milestone-table-add-missing-ids~ assigns incrementing IDs to rows that lack one.
- Feature: ~org-milestone-table-sort-by-date~ sorts data rows by date ascending; undated rows go to the end.
- Feature: Column label now accepts ~Pred~ or ~Predecessors~ (case-insensitive).
- Bugfix: Wrap ~omt--resolve~ in ~cl-block~ so ~cl-return-from~ works for cycle detection.
- Chore: Add autoload cookies, fix ~defgroup~ URL, silence byte-compiler warnings.
- CI: Add ~package-lint~ job for MELPA compliance checks.

* <2026-03-26 Mon>
- Chore: Rename ~org-milestone-table-empty~ to ~org-milestone-table-new~.

* <2026-03-24 Mon>
- Feature: ~org-milestone-table-empty~ now includes a Status column in the template.

* <2025-03-23 Sun>
- Feature: ~C-c C-c~ in a milestone table now runs add-missing-IDs, update-dates, and sort-by-date in sequence.
- Implemented via ~org-milestone-table-dwim~ on ~org-ctrl-c-ctrl-c-hook~.
- Bugfix: wrap ~omt--resolve~ body in ~cl-block~ so ~cl-return-from~ works for cycle detection.
- Column label now accepts "Pred" or "Predecessors" (case-insensitive).
- New command: ~org-milestone-table-add-missing-ids~ — finds max existing numeric ID, assigns incrementing IDs to rows that lack one.
- New command: ~org-milestone-table-sort-by-date~ — sorts data rows by Date ascending; rows without dates go to the end.

* <2025-03-15 Sat>
- Initial version.
- Dynamic column detection: parses header row to find =ID=, =Pred=, =Date= by name (case-insensitive).
- No hardcoded heading: uses ~(org-table-begin)~ --- cursor just needs to be inside any table.
- Dynamic column regex: builds cell-replacement regex from ~col-date~ rather than hardcoding column position.
- Month unit (=m=) support in Pred (e.g. =6+1m=) with proper calendar arithmetic.
- Table realignment after updates via ~(org-table-align)~.
- Only =ID=, =Pred=, =Date= columns are required; other columns are ignored and preserved.
- Bare number in Pred (e.g. =6=) is now treated as =6+0d= --- copies the referenced row's date.
- Pred referencing an unknown ID now produces an explicit error: ="Pred references unknown ID 'N'"=.
- Switch date format from =M /DD= to ISO 8601 =YYYY-MM-DD=.
- ~parse-date~ now matches =YYYY-MM-DD= pattern and extracts year from the string.
- ~format-date~ now emits zero-padded =YYYY-MM-DD=.
- Removed hardcoded year assumption (previously defaulted to 2025).
